# Remitos: COT, atributos y estado logístico

## Objetivo
Documentar los cambios mínimos de esquema requeridos para soportar paridad de remitos con `C:\Zulu\zuluApp` en PostgreSQL local (`localhost:5432`).

## Cambios cubiertos en código
- `estado_logistico` en `comprobantes`
- `es_valorizado` en `comprobantes`
- `deposito_origen_id` en `comprobantes`
- tabla `cmp_cot`
- tabla `comprobante_atributo`

## SQL sugerido
```sql
ALTER TABLE comprobantes
    ADD COLUMN IF NOT EXISTS estado_logistico varchar(30),
    ADD COLUMN IF NOT EXISTS es_valorizado boolean NOT NULL DEFAULT true,
    ADD COLUMN IF NOT EXISTS deposito_origen_id bigint;

CREATE INDEX IF NOT EXISTS ix_comprobantes_estado_logistico
    ON comprobantes (estado_logistico);

CREATE INDEX IF NOT EXISTS ix_comprobantes_deposito_origen_id
    ON comprobantes (deposito_origen_id);

CREATE TABLE IF NOT EXISTS cmp_cot (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    cmp_id bigint NOT NULL,
    cot_valor character varying(50) NOT NULL,
    cot_fecha date NOT NULL,
    cot_descripcion character varying(500),
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now(),
    deleted_at timestamptz,
    created_by bigint,
    updated_by bigint,
    CONSTRAINT fk_cmp_cot_comprobante
        FOREIGN KEY (cmp_id) REFERENCES comprobantes (id) ON DELETE CASCADE
);

CREATE UNIQUE INDEX IF NOT EXISTS ux_cmp_cot_cmp_id
    ON cmp_cot (cmp_id);

CREATE INDEX IF NOT EXISTS ix_cmp_cot_cot_valor
    ON cmp_cot (cot_valor);

CREATE INDEX IF NOT EXISTS ix_cmp_cot_cot_fecha
    ON cmp_cot (cot_fecha);

CREATE TABLE IF NOT EXISTS comprobante_atributo (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    comprobante_id bigint NOT NULL,
    clave character varying(100) NOT NULL,
    valor character varying(500),
    tipo_dato character varying(50),
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now(),
    deleted_at timestamptz,
    created_by bigint,
    updated_by bigint,
    CONSTRAINT fk_comprobante_atributo_comprobante
        FOREIGN KEY (comprobante_id) REFERENCES comprobantes (id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS ix_comprobante_atributo_comprobante_id
    ON comprobante_atributo (comprobante_id);

CREATE UNIQUE INDEX IF NOT EXISTS ux_comprobante_atributo_comprobante_clave
    ON comprobante_atributo (comprobante_id, clave);
```

## Validaciones funcionales de `zuluApp`
- `cot_valor` obligatorio
- longitud mínima `6`
- `cot_fecha >= comprobantes.fecha`
